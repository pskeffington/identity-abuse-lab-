# Safe Threat Model

## Purpose

Define a public-safe, defensive threat model for identity-abuse and misuse-risk evaluation.

## Current status

New public repository / intake needed.

## Defensive framing

This repository should support defensive evaluation, safety analysis, and public-safe research infrastructure. It should not include instructions, workflows, or examples that enable impersonation, fraud, credential misuse, evasion, or abuse.

## In-scope categories

- Defensive taxonomy of identity-abuse risks
- Public-safe evaluation criteria
- Evidence-boundary documentation
- High-level misuse-risk patterns without operational detail
- Defensive detection, audit, and governance controls
- Synthetic or toy examples that cannot enable abuse

## Out-of-scope categories

- Private identifiers or personal data
- Credential collection, credential misuse, or evasion workflows
- Impersonation instructions
- Fraud-enabling procedures
- Operational abuse playbooks
- Targeting guidance
