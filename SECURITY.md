# Security Policy

## Scope

Identity Abuse Lab is a defensive synthetic research repository. Security reports should focus on repository safety, accidental sensitive-data exposure, unsafe dual-use content, dependency issues, or workflow vulnerabilities.

## Do not submit

Do not submit real credentials, tokens, production logs, victim data, exploit payloads, malware samples, or live phishing artifacts as part of a report.

## Reporting concerns

If you identify a safety or security concern, open a private security advisory if available, or contact the repository owner through GitHub.

Examples of relevant concerns:

- accidental secret exposure;
- committed production data;
- unsafe wording that becomes operational abuse guidance;
- dependency vulnerabilities;
- CI workflow weakness;
- rules or examples that appear to target real systems rather than synthetic telemetry.

## Project safety boundary

The repository should not include:

- credential collection;
- live phishing or vishing workflows;
- malware;
- exploit development;
- unauthorized access guidance;
- production logs;
- real user records;
- real customer data; or
- real incident-sensitive details.

## Supported versions

The current development branch is `main`. Release tags will be documented in `RELEASE_NOTES_v0.1.md` and future release files.
